## About

This repository documents the design and implementation of a reference Information Security Management System (ISMS) based on the ISO/IEC 27001:2022 standard.

The project serves both as a professional portfolio and as a practical learning exercise, demonstrating the complete lifecycle of designing an ISMS for a healthcare organization.

The documentation covers every stage of the implementation process, including governance, organizational context, scope definition, asset management, information classification, risk assessment, risk treatment, control selection, security policies, operational procedures, business continuity and continual improvement.

For educational purposes, **NYC Health + Hospitals** is used as the reference organization. Whenever public information is unavailable, reasonable assumptions are explicitly documented.

This repository **does not represent the actual infrastructure, security architecture, policies or controls of NYC Health + Hospitals**, nor should it be interpreted as official documentation.

---

## Project Goals

- Design a complete Information Security Management System (ISMS) following ISO/IEC 27001:2022.
- Document every phase of the implementation process.
- Develop reusable templates, policies and operational procedures.
- Maintain full traceability between business processes, assets, risks, controls and documentation.
- Build a professional cybersecurity portfolio focused on Governance, Risk and Compliance (GRC).

## Project Scope

The reference ISMS includes:

- ISMS governance and documentation
- Organizational context
- Physical, logical and organizational scope
- Information classification
- Asset inventory
- Risk assessment and treatment
- Statement of Applicability (SoA)
- Security controls
- Security policies
- Operational procedures
- Business Continuity and Disaster Recovery
- Continuous improvement
