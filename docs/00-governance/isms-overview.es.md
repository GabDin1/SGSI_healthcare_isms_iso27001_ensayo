# 1. Introducción

Este documento proporciona una visión general del Sistema de Gestión de Seguridad de la Información (SGSI) desarrollado como proyecto de referencia conforme a la norma ISO/IEC 27001:2022.

Su finalidad es presentar el propósito del proyecto, el contexto general, las normas utilizadas y la organización de la documentación que compone el SGSI.

---

# 2. Propósito

El objetivo de este proyecto es diseñar un Sistema de Gestión de Seguridad de la Información (SGSI) completo para una organización del sector sanitario siguiendo las buenas prácticas establecidas por la familia de normas ISO/IEC 27000.

Este repositorio tiene un doble propósito:

- Servir como portfolio profesional.
- Servir como proyecto educativo de referencia para el estudio de ISO/IEC 27001.

---

# 3. Organización de referencia

Para facilitar el desarrollo del proyecto se utiliza **NYC Health + Hospitals** como organización de referencia.

Toda la documentación ha sido elaborada utilizando únicamente información pública.

Cuando la información pública no resulta suficiente, se realizan hipótesis razonables que se documentan de forma explícita.

Este proyecto **no representa la infraestructura, políticas, procedimientos ni controles reales** de dicha organización.

---

# 4. Normativa de referencia

El SGSI se desarrolla tomando como referencia las siguientes normas y marcos de trabajo:

- ISO/IEC 27001:2022
- ISO/IEC 27002:2022
- ISO/IEC 27005
- NIST Cybersecurity Framework (como referencia complementaria)

---

# 5. Objetivos del SGSI

Los principales objetivos son:

- Proteger la confidencialidad de la información.
- Garantizar la integridad de los activos de información.
- Asegurar la disponibilidad de los servicios críticos.
- Gestionar los riesgos de seguridad de la información.
- Establecer controles de seguridad apropiados.
- Mantener un proceso de mejora continua.

# 8. Principios del proyecto

Durante el desarrollo del SGSI se aplican los siguientes principios:

- Enfoque basado en riesgos.
- Protección de la información como activo principal.
- Mejora continua.
- Documentación trazable.
- Reutilización de documentación.
- Hipótesis claramente identificadas.
- Transparencia en las decisiones de diseño.
