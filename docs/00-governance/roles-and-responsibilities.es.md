# 1. Objetivo

Este documento define los principales roles y responsabilidades relacionados con el Sistema de Gestión de Seguridad de la Información (SGSI).

Su finalidad es establecer una estructura clara de responsabilidades para garantizar la correcta planificación, implantación, operación, seguimiento y mejora continua del SGSI.

---

# 2. Alcance

Las responsabilidades descritas en este documento son aplicables a todos los procesos incluidos dentro del alcance del SGSI.

Los roles definidos representan funciones organizativas y no personas concretas.

---

# 3. Roles y responsabilidades

| Rol | Responsabilidades principales |
|------|-------------------------------|
| **Alta Dirección** | Aprobar el SGSI, proporcionar recursos, definir la estrategia y revisar periódicamente el desempeño del sistema. |
| **Responsable del SGSI (ISMS Manager)** | Coordinar la implantación y mantenimiento del SGSI, gestionar riesgos, supervisar la documentación y promover la mejora continua. |
| **Departamento de TI** | Administrar la infraestructura tecnológica, implementar controles técnicos, gestionar copias de seguridad, monitorizar sistemas y responder a incidentes técnicos. |
| **Responsables de Proceso** | Identificar los activos de información bajo su responsabilidad, colaborar en el análisis de riesgos y asegurar el cumplimiento de las políticas aplicables. |
| **Usuarios** | Utilizar los activos de información de forma segura, cumplir las políticas de seguridad y comunicar incidentes o comportamientos anómalos. |
| **Auditor Interno** | Evaluar la conformidad del SGSI con los requisitos establecidos, identificar oportunidades de mejora y elaborar los informes de auditoría. |
| **Proveedores y Terceros** | Cumplir los requisitos de seguridad establecidos en los contratos y acuerdos de servicio aplicables. |

---

# 4. Principios de asignación de responsabilidades

Las responsabilidades definidas en este documento se asignan conforme a los siguientes principios:

- Separación adecuada de funciones.
- Responsabilidad claramente definida.
- Autoridad suficiente para desempeñar las funciones asignadas.
- Rendición de cuentas.
- Cumplimiento de la normativa aplicable.
- Mejora continua del SGSI.

---

# 5. Revisión

Este documento deberá revisarse cuando:

- Se produzcan cambios organizativos relevantes.
- Se modifique el alcance del SGSI.
- Se incorporen nuevos procesos críticos.
- Se detecten mejoras durante auditorías o revisiones por la dirección.
