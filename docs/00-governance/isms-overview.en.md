# 1. Introduction

This document provides an overview of the Information Security Management System (ISMS) developed as a reference project based on ISO/IEC 27001:2022.

Its purpose is to introduce the project, describe its objectives, identify the applicable standards and explain how the documentation is organized.

---

# 2. Purpose

The purpose of this project is to design a complete Information Security Management System (ISMS) for a healthcare organization following the best practices defined by the ISO/IEC 27000 family of standards.

This repository has two primary objectives:

- Serve as a professional portfolio.
- Serve as an educational reference project for studying ISO/IEC 27001.

---

# 3. Reference Organization

This project uses **NYC Health + Hospitals** as the reference organization.

All documentation has been developed using publicly available information.

Whenever public information is insufficient, reasonable assumptions are made and explicitly documented.

This project **does not represent the actual infrastructure, policies, procedures or security controls** of the organization.

---

# 4. Reference Standards

The ISMS is primarily based on:

- ISO/IEC 27001:2022
- ISO/IEC 27002:2022
- ISO/IEC 27005
- NIST Cybersecurity Framework (supporting reference)

---

# 5. ISMS Objectives

The main objectives are:

- Protect information confidentiality.
- Ensure information integrity.
- Maintain the availability of critical services.
- Manage information security risks.
- Implement appropriate security controls.
- Support continual improvement.

---

# 6. Documentation Scope

The project documentation includes:

# 8. Project Principles

The ISMS has been designed according to the following principles:

- Risk-based approach.
- Information as the primary asset.
- Continual improvement.
- Full documentation traceability.
- Reusable documentation.
- Explicit assumptions.
- Transparency in design decisions.
