# 1. Purpose

This document describes the reference technologies used within the Information Security Management System (ISMS) architecture.

Its purpose is to provide an overview of the primary platforms, services and technologies supporting the reference organization's infrastructure.

---

# 2. Scope

This document applies to the technologies supporting the business processes included within the ISMS scope.

The selected technologies represent a reference implementation based on industry best practices and do not necessarily reflect the actual infrastructure of NYC Health + Hospitals.

---

# 3. Technology Categories

The technologies are grouped into the following categories:

- Infrastructure.
- Networking.
- Identity Management.
- Operating Systems.
- Virtualization.
- Databases.
- Storage.
- Cloud Services.
- Security.
- Monitoring.
- Backup.

---

# 4. Reference Technologies

| Category | Reference Technology | Purpose |
|----------|----------------------|---------|
| Virtualization | VMware vSphere | Virtual infrastructure |
| Operating Systems | Windows Server / Ubuntu Server | Infrastructure and application services |
| Identity Management | Active Directory | Authentication and identity management |
| DNS / DHCP | Windows Server | Network services |
| Email | Microsoft Exchange Online | Corporate email |
| Collaboration | Microsoft 365 | Productivity and collaboration |
| Storage | SAN / NAS | Enterprise storage |
| Databases | Microsoft SQL Server / PostgreSQL | Data persistence |
| Firewall | Palo Alto Networks | Perimeter protection |
| VPN | IPSec / SSL VPN | Secure remote access |
| Endpoint Protection | Microsoft Defender for Endpoint | Endpoint security |
| SIEM | Microsoft Sentinel | Security monitoring and event correlation |
| Backup Management | Veeam Backup & Replication | Backup and recovery |
| Cloud Platform | Microsoft Azure | Authorized cloud services |

---

# 5. Selection Criteria

The reference technologies have been selected based on:

- Technology maturity.
- Widespread enterprise adoption.
- Security capabilities.
- Scalability.
- High availability.
- Integration capabilities.
- Vendor support.

---

# 6. Lifecycle Management

Technologies included within the ISMS scope shall be managed throughout their lifecycle, including:

- Evaluation.
- Acquisition.
- Deployment.
- Operation.
- Maintenance.
- Updates.
- Replacement.
- Secure decommissioning.
