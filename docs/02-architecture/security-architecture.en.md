# 1. Purpose

This document describes the reference security architecture used in the Information Security Management System (ISMS) project.

Its purpose is to define the main protection layers, security capabilities and architectural principles used to protect the information assets included within the ISMS scope.

---

# 2. Scope

The security architecture applies to all systems, services, infrastructure and information assets included within the ISMS scope.

It covers both on-premises environments and cloud-based services.

---

# 3. Design Principles

The security architecture is based on the following principles:

- Defense in Depth.
- Zero Trust as the reference security model.
- Principle of Least Privilege.
- Separation of duties.
- Security by Design.
- Secure by Default.
- Risk-based management.
- Continual improvement.

---

# 4. Security Layers

The architecture implements multiple protection layers.

## Physical Security

Protection of facilities, data centers and critical equipment through physical access controls and monitoring.

## Network Security

Perimeter protection, network segmentation, traffic filtering and communication monitoring.

## Identity and Access Management

Authentication, authorization and identity lifecycle management.

## System Security

Protection of servers, workstations and devices through secure configurations, software updates and system hardening.

## Information Protection

Information classification, encryption, key management and protection against unauthorized access.

## Monitoring and Incident Response

Event logging, continuous monitoring and security incident management.

---

# 5. Security Capabilities

The architecture incorporates capabilities for:

- Identity and Access Management (IAM).
- Multi-Factor Authentication (MFA).
- Privileged access management.
- Endpoint protection.
- Email security.
- Malware protection.
- Threat detection and incident response.
- Vulnerability management.
- Backup and recovery.
- Security logging and monitoring.

---

# 6. Security Objectives

The architecture is designed to:

- Protect information confidentiality.
- Ensure data integrity.
- Maintain the availability of critical services.
- Reduce the attack surface.
- Detect anomalous activities.
- Support incident response.
- Meet applicable legal and regulatory requirements.

---

# 7. Relationship with the ISMS

The security architecture provides the technical foundation for:

- Security control implementation.
- Risk assessment.
- Information asset protection.
- Incident management.
- Business continuity.
- Continual improvement of the ISMS.
