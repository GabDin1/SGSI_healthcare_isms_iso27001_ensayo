# 1. Objetivo

Este documento describe la organización de referencia utilizada para el diseño del Sistema de Gestión de Seguridad de la Información (SGSI).

Su finalidad es proporcionar el contexto organizativo necesario para el resto de la documentación del proyecto, incluyendo la definición del alcance, la identificación de activos, el análisis de riesgos y la selección de controles de seguridad.

---

# 2. Organización de referencia

El proyecto toma como referencia a **NYC Health + Hospitals**, la red pública de atención sanitaria de la ciudad de Nueva York.

La organización se utiliza únicamente como caso de estudio debido a la disponibilidad de información pública sobre su actividad, estructura y servicios.

Toda la documentación elaborada en este repositorio constituye una implementación de referencia y **no representa la infraestructura, políticas, procedimientos ni controles reales** de NYC Health + Hospitals.

Cuando la información pública no resulta suficiente, las hipótesis utilizadas se documentan de forma explícita.

---

# 3. Misión

Proporcionar servicios sanitarios accesibles, seguros y de calidad a la población, garantizando la protección de la información clínica y la continuidad de los servicios asistenciales.

---

# 4. Visión

Disponer de un Sistema de Gestión de Seguridad de la Información que permita proteger los activos de información críticos, reducir los riesgos de seguridad y apoyar la continuidad de los servicios sanitarios.

---

# 5. Actividad principal

La organización desarrolla actividades relacionadas con la prestación de servicios sanitarios, entre ellos:

- Atención hospitalaria.
- Atención ambulatoria.
- Servicios de urgencias.
- Diagnóstico por imagen.
- Laboratorios clínicos.
- Farmacia hospitalaria.
- Gestión administrativa.
- Investigación médica.

---

# 6. Infraestructura de referencia

Para este proyecto se considera una infraestructura híbrida compuesta por:

## Infraestructura local (On-Premises)

- Hospitales.
- Centros ambulatorios.
- Laboratorios.
- Centros administrativos.
- Centros de datos.
- Redes corporativas.
- Salas de comunicaciones.

## Servicios en la nube

- Correo electrónico corporativo.
- Plataformas de colaboración.
- Copias de seguridad externas.
- Recuperación ante desastres.

---

# 7. Información crítica

La organización gestiona información de alta sensibilidad, incluyendo:

- Historias clínicas electrónicas.
- Datos personales de pacientes.
- Resultados de laboratorio.
- Diagnósticos médicos.
- Prescripciones.
- Información financiera.
- Información de empleados.

La protección de esta información constituye el principal objetivo del SGSI.

---

# 8. Supuestos del proyecto

Este proyecto se desarrolla bajo las siguientes hipótesis:

- Se utiliza únicamente información pública.
- No se dispone de documentación interna.
- La arquitectura técnica es una implementación de referencia.
- Las decisiones de diseño se basan en buenas prácticas internacionales.
- Todas las hipótesis relevantes quedan documentadas.

---

# 9. Relación con el SGSI

La información descrita en este documento sirve como base para:

- Identificar las partes interesadas.
- Definir el alcance del SGSI.
- Identificar los procesos de negocio.
- Elaborar el inventario de activos.
- Realizar el análisis de riesgos.
