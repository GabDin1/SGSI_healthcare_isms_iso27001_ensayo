# 1. Purpose

This document describes the reference organization used for the design of the Information Security Management System (ISMS).

Its purpose is to provide the organizational context required for the remaining ISMS documentation, including scope definition, asset identification, risk assessment and security control selection.

---

# 2. Reference Organization

This project uses **NYC Health + Hospitals** as the reference organization.

The organization is used solely as a case study because of the availability of public information regarding its structure, operations and healthcare services.

All documentation contained in this repository represents a reference implementation and **does not represent the actual infrastructure, policies, procedures or security controls** of NYC Health + Hospitals.

Whenever public information is unavailable, reasonable assumptions are explicitly documented.

---

# 3. Mission

Provide accessible, safe and high-quality healthcare services while protecting sensitive information and ensuring the continuity of healthcare operations.

---

# 4. Vision

Maintain an Information Security Management System capable of protecting critical information assets, reducing information security risks and supporting the continuity of healthcare services.

---

# 5. Core Business Activities

The organization provides healthcare services including:

- Hospital care.
- Outpatient care.
- Emergency services.
- Medical imaging.
- Clinical laboratories.
- Hospital pharmacy.
- Administrative services.
- Medical research.

# 6. Reference Infrastructure

This project assumes a hybrid infrastructure consisting of:

## On-Premises Infrastructure

- Hospitals.
- Outpatient clinics.
- Clinical laboratories.
- Administrative offices.
- Data centers.
- Corporate networks.
- Communication rooms.

## Cloud Services

- Corporate email.
- Collaboration platforms.
- Off-site backups.
- Disaster recovery services.

---

# 7. Critical Information

The organization manages highly sensitive information, including:

- Electronic Health Records (EHR).
- Patient personal data.
- Laboratory results.
- Medical diagnoses.
- Prescriptions.
- Financial information.
- Employee information.

Protecting this information is the primary objective of the ISMS.

---

# 8. Project Assumptions

The project is developed under the following assumptions:

- Only publicly available information is used.
- No internal documentation is available.
- The technical architecture represents a reference implementation.
- Design decisions follow internationally recognized best practices.
- All relevant assumptions are explicitly documented.
